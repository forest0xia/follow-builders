**Claude Blog: New in Claude Managed Agents — dreaming, outcomes, and multiagent orchestration**

Anthropic announces three major updates to Claude Managed Agents. First, "dreaming" (research preview) is a scheduled process that reviews past agent sessions, extracts patterns, and refines memory stores so agents self-improve between sessions without human steering. Memory captures what each agent learns as it works; dreaming refines that memory, surfaces shared learnings across agents, and keeps it high-signal as it evolves. Second, "outcomes" lets developers write a success rubric; a separate grader evaluates the agent's output in its own context window and sends the agent back to iterate if something's off. In testing, outcomes improved task success by up to 10 percentage points — with the largest gains on the hardest problems — and boosted file generation quality by +8.4% on docx and +10.1% on pptx. Third, multiagent orchestration allows a lead agent to break complex jobs into pieces and delegate each to a specialist with its own model, prompt, and tools. Developers can now also set webhook notifications for when tasks complete.
https://claude.com/blog/new-in-claude-managed-agents

Anthropic 宣布了 Claude Managed Agents 的三项重要更新。一，"dreaming"（研究预览版）：一个定期审查过去 agent session、提取模式并优化记忆库的计划进程，让 agent 在无需人工干预的情况下持续自我改进。记忆捕捉每个 agent 工作中学到的内容，dreaming 则在 session 之间精炼这些记忆，提取跨 agent 的共同经验并保持高信噪比。二，"outcomes"：开发者编写成功标准，一个独立的 grader 在自己的上下文窗口中评估 agent 输出，有不足则让 agent 重新迭代。测试中 outcomes 将任务成功率提升最多 10 个百分点，最难问题收益最大，文件生成质量也有提升：docx +8.4%，pptx +10.1%。三，multiagent orchestration 允许主 agent 将复杂任务拆解并分配给各自拥有独立模型、prompt 和工具的专家 agent。开发者现在还可以设置任务完成时的 webhook 通知。

**The MAD Podcast with Matt Turck — State of Enterprise AI 2026: Aaron Levie on Tokenmaxxing, Rise of Headless, and AI-Proofing Your Job**

The Takeaway: AI model progress is paradoxically slowing enterprise deployment — because every breakthrough makes the last architecture obsolete before anyone finishes rolling it out.

Box CEO Aaron Levie sits down with Firstmark's Matt Turck to diagnose where large enterprises actually stand with AI in 2026. Levie's vantage point is rare: he runs a public company serving Fortune 500 and Global 2000 clients and talks to hundreds of CIOs each year.

The mood inside big enterprises is "remarkably optimistic," he says — not because hype has returned, but because CIOs are watching their engineering teams get genuinely faster with tools like Claude Code, Cursor, and Codex, and the business side is now pulling IT to get the same gains. The question is no longer "does this work?" It's "how do we roll this out to non-engineering knowledge work?"

The dominant conversation at every enterprise right now is token costs. Levie explains the structural shift: the old model of "include AI in the $20/seat subscription" breaks down when a single coding agent task can consume $1,000 of compute. Token budgets are moving out of IT spend and into line-of-business budgets — but marketing and sales teams don't have FinOps practices, and nobody has tooling yet to measure ROI on tokens. "I actually have no solutions," Levie says bluntly.

His broader framing: AI breakthroughs are compressing a normal ten-year compute cycle into twelve months, which means economies of scale haven't kicked in, pricing power stays with the labs, and every deployment plan keeps getting made obsolete. "The technology is getting so advanced that it makes obsolete the prior thing that you implemented — which actually means that the rollout takes longer."
