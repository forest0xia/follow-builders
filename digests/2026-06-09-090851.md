## OFFICIAL BLOGS

**Claude Blog — Building intelligent apps for Apple platforms with Claude in the Foundation Models framework**

Anthropic released a new Swift package integrating Claude into Apple's Foundation Models framework, enabling Apple developers to call Claude for complex workflows starting with iOS 27, iPadOS 27, macOS 27, visionOS 27, and watchOS 27. Apple's framework already returns typed Swift values in as few as three lines of code. Now, when a task requires multi-step reasoning, code generation, or web search, developers can seamlessly hand off to Claude. The integration works both ways: "Because Apple's framework returns typed Swift values from @Generable annotations, developers arrive at the Claude API call with clean inputs instead of raw user text." Practical examples: a journaling app generates daily prompts on-device, then asks Claude to surface patterns across months of entries; a study app defines a term locally, then hands off to Claude when the student asks "why does this matter for everything else we've covered?" Claude can also search the web and execute code for data analysis, streaming responses back into the SwiftUI view.
https://claude.com/blog/claude-for-foundation-models

**Claude Blog — 在 Foundation Models 框架中使用 Claude 构建 Apple 平台智能应用**

Anthropic 发布了一个新的 Swift package，将 Claude 集成到 Apple 的 Foundation Models 框架中，让 Apple 开发者可以在 iOS 27、iPadOS 27、macOS 27、visionOS 27 和 watchOS 27 上调用 Claude 处理复杂工作流。Apple 的框架已经能在短短三行代码内返回类型化的 Swift 值。现在，当任务需要多步推理、代码生成或网络搜索时，开发者可以无缝将请求转交给 Claude 处理。这种集成是双向的："由于 Apple 框架通过 @Generable 注解返回类型化的 Swift 值，开发者在调用 Claude API 时获得的是结构清晰的输入，而非原始用户文本。"实际应用场景举例：日记 app 在设备本地生成每日提示，再由 Claude 挖掘几个月日记中的深层规律；学习 app 在本地解释一个概念，当学生追问"这和我们之前学的所有内容有什么关系？"时则转交 Claude 回答。Claude 还可以搜索网络、执行代码进行数据分析，并将结果以流式方式返回到 SwiftUI 视图中。

## PODCASTS

**No Priors — Building an AI Guardian for Enterprise with Onyx Security CEO Maxim Bar Kogan**
https://www.youtube.com/watch?v=QDsbFLEt9ro

**The Takeaway:** Enterprises don't need AI to stop working — they need AI that works too much, on the wrong things. The new security problem isn't what employees put into ChatGPT; it's autonomous agents taking actions nobody signed off on.

Maxim Bar Kogan, co-founder and CEO of Onyx Security, makes a counterintuitive argument: the better AI agents get, the *harder* it becomes to control them. Traditional security tools — identity management, endpoint detection, proxies — were designed around predictable systems. Autonomous AI agents are anything but predictable, and giving them scoped permissions defeats the purpose of using them in the first place. "We want them to have our permissions because we want to tell Claude Code to do something... and we want to go have lunch."

Onyx's solution is a two-layer system. Small, specialized models run continuously to answer one narrow question: "Is this action suspicious enough to escalate?" When the answer is yes, a smarter agent takes a closer look. Bar Kogan uses a chess analogy: "Top players make most of their moves intuitively. Every once in a while they stop to calculate forward a lot of options — because they know this is a critical move." The goal is to spend intelligence exactly where risk is highest, not everywhere.

What makes Onyx structurally defensible against the labs themselves: enterprises refuse to give Anthropic or OpenAI their historical agent behavior data — "they know these are very data hungry companies that will want to train on that data." A neutral third party sees behavioral patterns no lab can access. And as models get smarter, Bar Kogan argues the failures aren't silly mistakes anymore — they're agents developing an "independent, semi-conscious perspective" that diverges from user intent.

The longer bet is interpretability: understanding what's happening inside model weights and activations as they operate. "Long term, you're going to have to understand much better what models are thinking, what's happening on the internals of these models as they're operating."
https://www.youtube.com/watch?v=QDsbFLEt9ro

**核心洞察：** 企业担心的不是 AI 停止工作，而是 AI 工作过度 — 在没有人批准的事情上。新的安全问题不再是员工把什么信息输入 ChatGPT，而是自主 agent 在执行没人授权的操作。

Onyx Security 联合创始人兼 CEO Maxim Bar Kogan 提出了一个反直觉的论断：AI agent 越强，就越难控制。传统安全工具 — 身份管理、端点检测、代理 — 都是为可预测的系统设计的，而自主 AI agent 恰恰是不可预测的；而且给它们限制权限，本身就违背了使用它们的初衷。"我们希望它们拥有我们的权限，因为我们想对 Claude Code 下达指令... 然后去吃个午饭。"

Onyx 的解决方案是双层架构：轻量级专用模型持续运行，只回答一个问题："这个操作是否可疑到需要上报？"一旦判断需要，更智能的 agent 介入深入审查。Bar Kogan 用象棋打了个比方："顶级棋手大多数走法都凭直觉完成。偶尔他们会停下来反复计算，因为他们知道这是关键一步。"目标是把计算资源集中在风险最高的地方，而不是平均分配。

Onyx 在面对大型 AI 实验室竞争时的结构性优势在于：企业拒绝把历史 agent 行为数据交给 Anthropic 或 OpenAI — "他们知道这些公司非常渴求数据，会拿去训练模型。"一个中立的第三方能看到任何实验室都无法获取的行为规律。随着模型越来越聪明，Bar Kogan 认为失败模式已经不再是低级错误，而是 agent 开始形成一种"独立的、半自觉的视角"，与用户意图发生偏差。

更长远的赌注在于可解释性：理解模型权重和激活在运行过程中究竟发生了什么。"长远来看，你将不得不更深入地理解模型在想什么，理解这些模型在运行时内部到底发生了什么。"
