**Latent Space — "Extreme Harness Engineering for Token Billionaires: 1M LOC, 1B toks/day, 0% human code, 0% human review"**
https://www.youtube.com/@LatentSpacePod

**The Takeaway:** When your team stops writing code and starts engineering the harness, your bottleneck shifts from model capability to human attention — and that changes everything about how software is built.

Ryan Lopopolo, who works on frontier product exploration at OpenAI Frontier (the enterprise agent deployment platform), joins Swyx and Alessio to walk through an extraordinary experiment: building a 1-million-line production codebase over five months with zero lines of human-written code, over 1,500 PRs, and post-merge human review only. His background spans Stripe, Brex, Snowflake, and Citadel — exactly the enterprise customer OpenAI wants to win over.

The key insight: Ryan and a team of three constrained themselves from writing any code so they could force the agent to become genuinely autonomous. The first six weeks were 10x slower — painful and disorienting. But by investing in the right abstractions, tooling, and documentation early, they eventually built an "assembly station" so effective the agent could handle the entire SDLC: code, tests, CI configuration, release tooling, documentation, code review comments, Grafana dashboard JSON, even responding to production pages.

A few standout lessons from the conversation:
- **Build time discipline is agent discipline.** They retooled their entire build system to complete under one minute — because if the agent has to wait too long, it becomes less patient and less effective. Cheap tokens make it economically viable to constantly "garden" these invariants.
- **Scaffold less, context more.** Reasoning models don't need rigid scaffolds. Instead, give them rich context (AGENT.md, a short table of contents, skills) and let them reason about what to do next. Ryan effectively reinvented "skills" from first principles before OpenAI released the feature.
- **Human attention is the scarce resource.** The agent is trivially parallelizable. Ryan now sees himself as "group tech leading a 500-person org" — setting architectural principles and monitoring for patterns, not reviewing every PR.
- **Everything is prompt injection.** Docs, tests, error messages, code review agents — all of it is a mechanism for encoding what "good" looks like so the model self-corrects. When the agent makes a mistake, that's a signal of an undocumented non-functional requirement.
- **On internalizing dependencies:** Brett Taylor's observation that software dependencies might go away (just vendor them with the model) got partial agreement — Ryan notes a "couple-thousand-line dependency" can be in-housed in an afternoon, stripping away unused generics. But for larger-scale or security-critical dependencies, the external open source community still has the edge.

This episode is required listening for anyone building agentic software systems at scale.

Latent Space 播客 — 《为 Token 亿万富翁们的极端 Harness 工程：100万行代码、每天10亿token、0%人工代码、0%人工审查》
https://www.youtube.com/@LatentSpacePod

**核心观点：** 当你的团队停止写代码、转而专注于"harness 工程"时，瓶颈就从模型能力转移到了人类注意力——而这从根本上改变了软件的构建方式。

Ryan Lopopolo 在 OpenAI Frontier（企业级 agent 部署平台）负责前沿产品探索，他与 Swyx 和 Alessio 一起详细介绍了一项非凡的实验：用五个月时间，在没有一行人工代码、超过 1500 个 PR、且只在合并后进行人工审查的情况下，构建了一个拥有 100 万行代码的生产级代码库。他的背景横跨 Stripe、Brex、Snowflake 和 Citadel——正是 OpenAI 最希望赢得的企业客户画像。

核心洞察：Ryan 和三人小团队给自己设定了"不写任何代码"的约束，以此倒逼 agent 实现真正的自主性。前六周慢了10倍——痛苦且迷失方向。但通过早期投入正确的抽象、工具和文档，他们最终构建出了一个极其高效的"装配站"，让 agent 能够处理整个软件开发生命周期：代码、测试、CI 配置、发布工具、文档、代码审查评论、Grafana 看板 JSON，甚至响应生产告警。

对话中的几个关键经验：
- **构建时间纪律 = Agent 纪律。** 他们将整个构建系统改造为在一分钟内完成——因为等待时间太长会让 agent 变得不耐烦、效率下降。而廉价的 token 使得持续"维护"这些不变量在经济上完全可行。
- **少用脚手架，多给上下文。** 推理模型不需要刚性的脚手架。相反，给它们丰富的上下文（AGENT.md、简短目录、skills），让模型自己推理下一步。Ryan 在 OpenAI 正式推出"skills"功能之前，已经从零独立"重新发明"了它。
- **人类注意力是稀缺资源。** Agent 可以无限并行扩展；人不行。Ryan 现在把自己的角色定位为"统领500人团队的首席技术负责人"——制定架构原则、监察规律性问题，而不是亲自审查每一个 PR。
- **一切皆 prompt 注入。** 文档、测试、报错信息、代码审查 agent——所有这些都是将"什么是好的代码"编码进去的机制，让模型实现自我纠正。每当 agent 犯错，那就意味着某个非功能性需求尚未被文档化记录。
- **关于内化依赖项：** Brett Taylor 认为软件依赖项可能会消失（用模型直接集成进来）的观点获得了部分认可——Ryan 表示，一个几千行的依赖项可以在一下午内完成内化，同时剥去所有不需要的通用逻辑。但对于更大规模或安全敏感的依赖项，外部开源社区仍然具有优势。

这期节目是所有在规模化场景下构建 agentic 软件系统的人的必听内容。
