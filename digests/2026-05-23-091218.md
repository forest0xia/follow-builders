**Claude Blog: New in Claude Managed Agents — dreaming, outcomes, and multiagent orchestration**

Anthropic launched three major updates to Claude Managed Agents. The headline is "dreaming" — a scheduled process that reviews past agent sessions and memory stores, extracts recurring patterns, and curates memories so agents self-improve over time. Dreaming operates at a level no single agent can: it surfaces mistakes that repeat across runs, workflows agents naturally converge on, and preferences shared across a team. Developers control the autonomy level — dreaming can update memory automatically or queue changes for human review.

The second update, "outcomes," lets developers write a rubric defining what success looks like. A separate grader evaluates each output against that rubric in its own context window, then directs the agent to take another pass on what fell short. In testing, outcomes improved task success by up to 10 points over standard prompting, with the largest gains on the hardest problems. File generation quality also jumped: +8.4% on .docx and +10.1% on .pptx. Third, multiagent orchestration is now available: when a task exceeds what one agent can do well, a lead agent breaks the work into pieces and delegates each to a specialist with its own model, prompt, and tools. Webhooks complete the picture — define an outcome, let the agent run, get notified when it's done.

https://claude.com/blog/new-in-claude-managed-agents

Anthropic 为 Claude Managed Agents 推出了三项重要更新。最引人注目的新功能是"dreaming"（梦境）——一个定期审查历史 agent 会话和记忆库的调度进程，提取反复出现的模式，精炼记忆，使 agent 能够随时间自我改进。Dreaming 工作在单个 agent 无法触达的层面：发现跨多次运行重复出现的错误、agent 自然收敛的工作流，以及团队中共同的偏好。开发者可控制 dreaming 的自主程度——可以自动更新记忆，也可以先将变更排队等待人工审核。

第二项更新"outcomes"（目标）允许开发者编写描述成功标准的评分规则，系统用一个独立的 grader 在其自己的 context window 中评估输出，再告诉 agent 需要改进哪些地方。测试中，outcomes 在标准 prompting 循环基础上将任务成功率提升最多 10 个百分点，在最难的问题上提升幅度最大；文件生成质量也有所提升：.docx 格式 +8.4%，.pptx 格式 +10.1%。第三，multiagent orchestration 现已开放：当任务量超过单个 agent 的处理能力时，主 agent 可将任务拆分并分配给各专职 agent，每个 agent 拥有独立的模型、prompt 和工具。Webhook 支持则让开发者可在任务完成时收到通知。

**Unsupervised Learning — Ep 87: Gemini Co-Lead on World Models, RL's Next Domains & Continual Learning**

**The Takeaway:** The path to more capable AI agents is an iterative loop — model capability enables better systems, and better systems unlock the next level of model capability. There's no single breakthrough, just compounding cycles.

Oriol Vinyals, co-lead of Gemini at Google alongside Noam Shazir and Jeff Dean, has spent over a decade building systems that go beyond language to understand the visual world. His team's conviction has been consistent since before Gemini: "Since day one and way before the Gemini program started, we were working on not just language, but understanding the visual world and jointly modeling words in the context of vision, image, video."

Vinyals argues we haven't yet seen the "GPT moment" for video and images — that inflection where purely visual training transfers as powerfully as language pretraining has. The fundamental challenge is extracting conceptual knowledge from raw video without text labels. You can describe gravity in text, but can a model *infer* it from video alone, without language scaffolding? Vinyals thinks answering that would be massive, but it remains research-stage. The recipe today is mixing all modalities together — which has produced impressive capabilities in Omni — but pure unsupervised visual transfer is still an open quest.

On agents, the step change visible in Spark (Google's new consumer agent at IO) came from a sequencing of releases rather than any single insight. Get the model good, build the system around it, then co-optimize both. The key architectural philosophy: keep the model general, and let specialization happen through intelligence layering rather than bespoke builds. On memory, Vinyals draws a direct parallel to reasoning improvements over the past few years — quiet, compounding progress that only looks obvious in retrospect.

https://www.youtube.com/watch?v=NQczevdpxq0

**要点：** 通往更强大 AI agent 的路径是一个迭代循环——模型能力提升使更好的系统成为可能，更好的系统又反过来支撑下一代模型能力的涌现。没有单一的突破，只有持续复利的循环。

Oriol Vinyals 是 Google Gemini 的共同负责人，与 Noam Shazir 和 Jeff Dean 并列，职业生涯超过十年，致力于构建不仅限于语言、能理解视觉世界的系统。他团队的信念在 Gemini 启动前就已形成："从第一天起，甚至早在 Gemini 项目启动之前，我们的工作就不只是语言，而是理解视觉世界，在视觉、图像、视频等多模态背景下联合建模。"

Vinyals 认为，视频和图像领域的"GPT 时刻"尚未到来——那个纯视觉训练能像语言预训练一样强力迁移的拐点还没有出现。核心挑战在于：如何在没有文本标签的情况下，从原始视频中提取概念性知识。文字可以描述重力，但模型能否仅凭视频、无需语言脚手架就推断出重力？Vinyals 认为一旦解决影响将是颠覆性的，但目前仍处于研究阶段。当前的做法是混合所有模态共同训练——Omni 已展示出令人印象深刻的能力——但纯无监督视觉迁移依然是一个开放性课题。

关于 agent：IO 上展示的 Google 新消费者 agent Spark 的能力跃升，来自一系列有序的迭代，而非某个单一洞见。先把模型做好，再在其上构建系统，最后联合优化。核心架构理念是在模型层保持通用性，通过智能分层实现专业化，而非构建专用系统。关于记忆，Vinyals 将其下一阶段的改进与近年来推理能力的提升直接类比——是一种不声不响的复利式进步，只有回望时才显得理所当然。
