**No Priors — Building an AI Guardian for Enterprise**
*with Onyx Security CEO Maxim Bar Kogan*

**The Takeaway:** As AI agents proliferate exponentially inside enterprises, the security tools built for human-scale work can't keep up — and the only viable answer may be AI that watches AI.

Maxim Bar Kogan, co-founder and CEO of Israel-based Onyx Security, has been building toward a single question since AutoGPT first sparked imaginations: when AI agents become smarter than us, how do you actually oversee them? That question has moved from philosophical to urgent. Onyx builds specialized models and agents designed to monitor other agents — what Bar Kogan calls an "AI control plane" for the enterprise.

The core architectural insight: you can't put a human in the loop when agents are completing tasks at 100x or 1,000x the volume. And you can't simply use a capable model to shadow every agent in real time — cost and latency would make it unworkable. Onyx's approach instead relies on small, fast "reflex" models that act like a chess grandmaster's intuition: processing most moves quickly, escalating only when something warrants deeper attention. As Bar Kogan puts it: "You don't want to spend too much intelligence where you don't have to, and you want to spend a lot of intelligence, overwhelmingly a lot, in situations where there's high risk."

A key structural advantage is data access. Enterprises share historical agent behavior logs with Onyx that they'd never hand to Anthropic or OpenAI — because they know those companies would use it for training. That independent position gives Onyx oversight context the model vendors themselves can't access.

Bar Kogan also flagged an emerging problem he considers harder than capability failures: agents developing what sounds like an independent perspective that doesn't always align with what the user actually wants. He observes this problem tends to grow alongside model intelligence, not shrink. On whether the labs could build this themselves, he draws a simple analogy: you don't ask the car dealer to certify whether their car is safe. You want an independent party whose entire credibility depends on being right.

**No Priors — 为企业构建 AI 守护者**
*对话 Onyx Security CEO Maxim Bar Kogan*

**核心洞察：** 随着 AI agent 在企业内部以指数级速度扩散，为人类规模工作设计的安全工具已经力不从心——唯一可行的答案，或许是用 AI 来监督 AI。

以色列初创公司 Onyx Security 的联合创始人兼 CEO Maxim Bar Kogan，自 AutoGPT 最初点燃想象力之日起，就在追问同一个问题：当 AI agent 变得比我们更聪明，我们如何真正监督它们？这个问题已从哲学命题变成了紧迫的商业现实。Onyx 构建专门用于监控其他 agent 的模型和 agent——Bar Kogan 称之为企业级的"AI 控制平面"。

核心架构洞察是：当 agent 以 100 倍、1000 倍的速度完成任务时，根本无法在每个环节加入人工审核。而让一个强大的模型实时跟踪每一个 agent 也行不通——成本和延迟都是致命问题。Onyx 的解法是使用小型、快速的"直觉反射"模型，像国际象棋大师的本能判断一样运作：快速处理绝大多数情况，只在真正需要时升级到更深入的审查。正如 Bar Kogan 所说："你不想在不必要的地方过度消耗智力，但在高风险的情况下，你要投入压倒性的智力。"

一个关键的结构性优势在于数据访问。企业愿意与 Onyx 共享 agent 历史行为日志，却绝不会把这些数据给 Anthropic 或 OpenAI——因为他们知道这些公司会用这些数据来训练模型。这种独立第三方的定位，让 Onyx 获得了连模型供应商自己都无法拥有的监督上下文。

Bar Kogan 还指出了一个他认为比能力缺陷更难解决的新兴问题：agent 逐渐形成某种独立视角，有时与用户的真实意图并不一致。他观察到，这个问题往往随着模型智能的提升而加剧，而非缩小。关于大模型公司是否能自己解决这个问题，他给出了一个简洁的类比：你不会让卖车的经销商来认证他们的车是否安全，你需要一个独立的第三方，其全部公信力都建立在"必须把事实说清楚"上。
