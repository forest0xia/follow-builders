### Claude Blog

**Claude Code now supports artifacts**

Claude Code can now turn session work into live, shareable visual pages called artifacts — including PR walkthroughs, system explainers, dashboards, and release checklists that refresh in real-time as your session progresses. Built from the full context of your session (codebase, connectors, and conversation), artifacts require no extra infrastructure: ask for a page, and Claude Code builds it from what already exists. When Claude Code republishes an artifact, the open page updates in place so every teammate sees the latest version at the same URL, with full version history for rollbacks.

Access is private by default — share with teammates from the page header. Organization admins control visibility with org-level toggles, role-based scoping, retention policies, and a compliance API. Artifacts is now in beta for Claude Team and Enterprise orgs, available from the Claude Code CLI and desktop app, with pages viewable in any browser.

Claude Code 现在可以将 session 的工作成果转化为实时、可共享的可视化页面（即 artifact），包括 PR 说明、系统架构解释、数据看板和发布清单，这些页面会随着 session 的推进自动刷新。Artifact 完全基于你的 session 上下文构建（代码库、连接器和对话内容），无需搭建任何额外基础设施：直接提出需求，Claude Code 就能从现有信息中生成页面。每次 Claude Code 重新发布 artifact，所有打开该页面的团队成员都能在同一 URL 看到最新版本，并可随时通过版本历史回滚。

访问权限默认为私有，可直接从页面顶部分享给团队成员。组织管理员可通过组织级开关、基于角色的权限范围、数据保留策略和 compliance API 统一管控访问权限。Artifacts 功能目前已面向 Claude Team 和 Enterprise 用户开放测试，支持通过 Claude Code CLI 和桌面 App 使用，生成的页面可在任意浏览器中查看。
