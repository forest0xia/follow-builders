### Anthropic Engineering

**An update on recent Claude Code quality reports**

Anthropic has disclosed a postmortem on three separate bugs that caused Claude Code quality degradation over March–April 2026, affecting Sonnet 4.6, Opus 4.6, and Opus 4.7. All three have been resolved as of April 20 (v2.1.116). The issues: (1) Claude Code's default reasoning effort was quietly downgraded from "high" to "medium" on March 4 to reduce latency — reverted April 7 after user backlash; (2) A bug introduced March 26 caused Claude to lose context every single turn after session idle periods, not just once as intended — fixed April 10; (3) A verbosity-reducing system prompt added April 16 hurt coding quality when combined with other prompt changes — reverted April 20. Because the three issues hit different user segments at different times, the aggregate effect looked like broad, inconsistent degradation that was hard to diagnose. As of April 23, Anthropic reset usage limits for all subscribers. The core lesson: internal evals failed to catch these regressions, so the team is now investing in tighter feedback loops between user reports and automated testing.

**Scaling Managed Agents: Decoupling the brain from the hands**

Anthropic Engineering explains the architectural philosophy behind Managed Agents — a hosted service that runs long-horizon agents on your behalf. The key insight: instead of coupling the session, harness, and sandbox into a single container (what they call "adopting a pet"), they decomposed them into independently swappable components, modeled after how operating systems virtualize hardware into stable abstractions. In their earlier monolithic design, a container failure meant the entire session was lost. By decoupling, each interface's implementation can be swapped without breaking the others — just as the read() syscall works identically whether you're reading a 1970s disk pack or a modern SSD. The post also flags a concrete anti-pattern: harnesses that encode Claude's limitations become dead weight as models improve. The "context anxiety" fix that worked on Sonnet 4.5 was completely unnecessary on Opus 4.5.

https://www.anthropic.com/engineering/managed-agents

Anthropic Engineering 阐释了 Managed Agents 背后的架构理念——这是一项代您运行长周期 agent 的托管服务。核心思路是：将 session（会话日志）、harness（编排循环）和 sandbox（执行环境）从单一容器中拆分出来，各自独立、可替换——就像操作系统将硬件虚拟化为稳定抽象接口一样。在早期的单体设计中，容器故障会导致整个 session 丢失。通过解耦，每个接口背后的实现可以自由替换，而不影响其他组件——正如 read() 系统调用无论面对 1970 年代的磁盘还是现代 SSD 都保持一致。文章还指出了一个具体的反模式：针对模型局限性硬编码的 harness 会随模型进步变成负担。在 Sonnet 4.5 上有效的"上下文焦虑"修复方案，在 Opus 4.5 上已完全不必要。

### Claude Blog

**New in Claude Managed Agents: self-hosted sandboxes and MCP tunnels**

Claude Managed Agents now supports two enterprise-critical features: self-hosted sandboxes (public beta) and MCP tunnels (research preview). Self-hosted sandboxes let enterprises keep tool execution within their own perimeter — on their infrastructure or with managed providers like Cloudflare, Daytona, Modal, or Vercel — while the orchestration layer stays on Anthropic's infrastructure. MCP tunnels enable agents to securely connect to private MCP servers inside an enterprise network. The announcement includes real production deployments: Amplitude's Design Agent uses Cloudflare for tighter observability and control; Clay's GTM engineering agent "Sculptor" runs on Daytona for long-running, stateful sessions. The core architectural move is separating where the model and context live (Anthropic) from where code executes and files are touched (your perimeter), so enterprise security policies, audit logging, and resource controls apply naturally.

https://claude.com/blog/claude-managed-agents-updates

### AI & I by Every

**We Automated Everything With AI and Tripled Our Headcount**

**The Takeaway:** Automation doesn't eliminate work — it multiplies the demand for human judgment, because AI produces floods of "almost right" output that only experts can properly evaluate and improve.

Every CEO Dan Shipper published an essay called "After Automation" arguing that heavy AI adoption leads to more human work, not less. Every's COO Brandon interviews Dan to probe the thesis against the wave of executive announcements about mass AI-driven layoffs. Every itself is a data point: the company has grown from 4 to 30 people since the GPT-3 era, while deploying AI agents throughout its operations — "if you swing a stick around in our Slack, you're as likely to hit a human as you are an agent."

Dan's core argument: AI makes "yesterday's expert competence cheap." When that competence becomes cheap, everyone adopts it — but the resulting output all looks similar, is trained on the same data, and is "close but not quite right" for any specific situation. The flood of almost-right output actually *increases* demand for two types of human work: (1) experts who build systems to route, evaluate, and improve the AI-generated work; (2) experts using AI as a force multiplier to build things previously impossible — like one Every engineer who built an entire inbox product end-to-end in a month. The episode's most quotable line: "The further away an agent gets from a human, the less valuable it is."
