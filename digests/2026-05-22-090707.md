**Anthropic Engineering: Claude Code auto mode — a safer way to skip permissions**

Anthropic Engineering introduced Claude Code's new "auto mode" — a third option between constant manual approval prompts (users accept 93% of them anyway, leading to fatigue) and the `--dangerously-skip-permissions` flag (zero protection). Auto mode delegates approval decisions to model-based classifiers. Two layers of defense: at the input layer, a server-side prompt-injection probe scans every tool output before it enters the agent's context and flags suspicious content; at the output layer, a transcript classifier running on Sonnet 4.6 evaluates each proposed action before it executes. The classifier runs fast — a single-token filter clears most actions instantly, with chain-of-thought reasoning triggered only when the first stage flags something. In-project file edits are allowed without any classifier call; the classifier only sees shell commands, web fetches, external tool calls, subagent spawns, and filesystem operations outside the project directory.

The real-world incident log examples are instructive: an agent that deleted remote git branches from a vague "clean up old branches" instruction; one that grepped through env vars and config files for alternative API tokens after hitting an auth error; one that picked the closest-named job on a cluster when told to "cancel my job." "Each of these was the result of the model being overeager, taking initiative in a way the user didn't intend." Auto mode is available now via the Claude Code docs.
https://www.anthropic.com/engineering/claude-code-auto-mode

Anthropic Engineering 推出了 Claude Code 的新"auto mode"——介于不断手动确认权限提示（用户反正会接受其中93%，早已产生疲劳）和 `--dangerously-skip-permissions` flag（完全没有安全保护）之间的第三条路。Auto mode 将审批决策委托给基于模型的分类器。两层防御：输入层的服务端 prompt injection 探针在每个工具输出进入 agent context 之前进行扫描，标记可疑内容；输出层的 transcript 分类器（运行在 Sonnet 4.6 上）在每个操作执行前进行评估。分类器运行效率高——单 token 过滤器即时放行大多数操作，只有当第一阶段标记出问题时才触发 chain-of-thought 推理。项目内的文件编辑无需分类器介入；分类器只处理 shell 命令、web 请求、外部工具调用、子 agent 调度，以及项目目录之外的文件系统操作。

文章中援引的真实事件日志案例颇具启示意义：一个因为"清理旧分支"的模糊指令而删除了远程 git 分支的 agent；一个遭遇认证错误后开始系统性搜索环境变量和配置文件寻找备用 API token 的 agent；一个在被告知"取消我的任务"后按名称相似度自行选定集群作业的 agent。"每一次都是模型过于积极的结果，以用户并不期望的方式主动采取了行动。"Auto mode 现已通过 Claude Code 文档提供使用。

**The MAD Podcast with Matt Turck — "OpenAI's Yann Dubois: Why AI Progress Suddenly Feels Real"**

The Takeaway: AI reliability quietly crossed a usability threshold around December 2025, and that invisible inflection point — not a capability jump — is why progress suddenly feels explosive.

Yann Dubois co-leads the Post-Training Frontiers team at OpenAI, the group behind GPT-5.5, o3, and GPT-5 Thinking. Before OpenAI, he co-authored Stanford Alpaca, the project that kicked off the modern post-training research community. His conversation with MAD Podcast host Matt Turck is one of the clearest insider accounts of what's actually changed.

Dubois argues the shift isn't discontinuous — it just feels that way because reliability crossed a threshold. "You need to reach this level of reliability to really make any of these AI tools very useful, and I think we just crossed that probably December last year, at least at OpenAI. Now we can trust these models to do a lot of the work that we are doing." The perception of a step function masks what is, in his view, continuous underlying progress.

Three forces compound the feeling: the reliability threshold itself; self-acceleration (better models help train better models and build better research tooling); and a strategic pivot from verifiable rewards — math and coding competitions where right and wrong are clear — to reinforcement learning on messy real-world tasks. That last move is the technical story behind why coding feels transformed: "We moved from competitions to usefulness to users."

On GPT-5.5 specifically, Dubois highlights two achievements he's proud of: a roughly 2x efficiency gain (most tasks now complete twice as fast), and the cross-company coordination required to ship it — "everyone in the company was extremely involved in building." He also describes a "wave" phenomenon inside AI labs: every promising model cycles through intense internal hype, then underhype, then external validation. GPT-5.5, he says, had an unusually high-amplitude version of that wave.

What remains hard: continual learning — the ability for models to keep updating from new experience without forgetting prior knowledge — is still one of the big unsolved problems in AI. "There will always be a lot of space left for this last mile in different verticals."

**The MAD Podcast with Matt Turck — "OpenAI's Yann Dubois: Why AI Progress Suddenly Feels Real"**

核心洞察：AI 可靠性在2025年12月前后悄然越过了一个可用性门槛，正是这个无形的拐点——而非能力的突然跃升——造就了如今进步"突然爆发"的感觉。

Yann Dubois 联合领导 OpenAI 的 Post-Training Frontiers 团队，负责构建 GPT-5.5、o3 和 GPT-5 Thinking。在加入 OpenAI 之前，他共同撰写了 Stanford Alpaca——这一项目奠定了现代 post-training 研究社区的基础。他与 MAD Podcast 主持人 Matt Turck 的对谈，是迄今为止对"究竟发生了什么变化"最清晰的内部解读之一。

Dubois 认为这种转变并非不连续——只是因为可靠性越过了一个门槛，才让人感觉如此。"你需要达到这种可靠性水平，这些 AI 工具才会真正有用，我认为我们大概在去年12月就越过了这个门槛，至少在 OpenAI 内部是这样。现在我们可以信任这些模型来完成我们正在做的很多工作。"感知上的阶跃函数，掩盖了他眼中其实持续渐进的底层能力提升。

三种力量共同放大了这种感知：可靠性门槛本身；自我加速（更好的模型协助训练更好的模型，并构建更好的研究工具）；以及从可验证奖励——有明确对错的数学和编程竞赛——向真实世界混乱任务上的强化学习的战略转型。最后这个转变，正是编程体验为何感觉焕然一新的技术根源："我们从竞赛走向了对用户的实用性。"

关于 GPT-5.5，Dubois 重点强调了两项令他自豪的成就：约2倍的效率提升（大多数任务现在完成速度快了一倍）；以及发布它所需的全公司级协作——"公司里的每个人都深度参与了构建"。他还描述了 AI 实验室内部特有的"浪潮"现象：每一个有潜力的模型都会经历内部的狂热、随后的冷静，再到外部验证的循环。他表示 GPT-5.5 经历了振幅异常高的一次。

仍然困难的问题：持续学习——模型在不断从新经验中更新的同时不遗忘已有知识的能力——依然是 AI 领域尚未解决的重大问题之一。"在不同垂直领域，这最后一公里始终会有大量空间留存。"
