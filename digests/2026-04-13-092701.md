**Box CEO Aaron Levie** returned from another week on the road with enterprise AI leaders across banking, healthcare, retail, and media, and the pattern he's seeing is clear: the "chat era" of AI is giving way to agents that actually execute work. His field notes are sharp:

- **Change management is the bottleneck.** One company has a head of AI in every business unit reporting to a central team. Enterprises can't just drop agents into existing workflows.
- **Tokenmaxxing is real.** Companies operating on fixed OpEx budgets are having serious internal debates about how to ration compute — one company pitched a "shark tank" model for compute allocation.
- **Engineers are not going away.** Despite Silicon Valley's confidence that AI makes hard things easy, enterprise automation requires technical people to set up and operate the systems. "Engineers may not be writing software, but they will certainly be the ones to setup and operate the systems that actually automate most work in the enterprise."

On security, Levie flagged the Jevons paradox in action: better AI security tooling will *increase* demand for security talent, not decrease it. More bugs found faster means more triage, more remediation, more human judgment needed at each decision point.

**Claude Blog: Preparing your security program for AI-accelerated offense**

Anthropic's security post lands as a companion piece to Project Glasswing — their initiative to deploy Claude Mythos Preview's vulnerability-finding capabilities for defense. The core warning: within the next 24 months, AI will expose vast numbers of latent bugs that have sat undetected in codebases for years and chain them into working exploits. The window between patch release and weaponized exploit is already shrinking.

The practical guidance is organized around what security teams should do right now:

1. **Close your patch gap** — Patch everything on the CISA Known Exploited Vulnerabilities (KEV) catalog immediately, treating anything internet-reachable as an emergency. Use EPSS scores to prioritize the rest. Automate patch deployment where outage risk is acceptable; manual approval steps are "delay, and delay is now the primary risk."

2. **Prepare for a flood of vulnerability reports** — Plan for an order-of-magnitude increase in finding volume from both your own codebase scans and upstream vendors. Intake, triage, and remediation tracking all need to scale.

The post is notable for its candor: Anthropic built a model that can find serious vulnerabilities, and they're now telling defenders exactly how much faster the threat landscape is about to move. The advice maps directly to SOC 2 and ISO 27001 controls, and the team has committed to updating guidance as Project Glasswing work continues.

**Latent Space: Marc Andreessen introspects on The Death of the Browser, Pi + OpenClaw, and Why "This Time Is Different"**

**The Takeaway:** AI agents aren't just a new kind of software — they're the Unix shell moment for the language model era, and Marc Andreessen argues the architecture is now obvious in retrospect: LLM + shell + file system + markdown + cron.

Marc Andreessen has been around long enough to have lived through multiple AI winters — he coded in Lisp in 1989. His argument for why this wave is different is structural: we now know, definitively, that the neural network is the correct architecture. That wasn't settled until recently, after a 70-year debate. Everything being built today draws from an original idea in a 1943 paper. "It's an eighty year overnight success," he says. "ChatGPT hits, o1 hits, OpenClaw hits — these are radical overnight transformative successes, but they're drawing on an eighty year Wellspring of ideas."

On the Pi and OpenClaw architectural breakthrough, Andreessen draws a direct line from the 1970s Unix mindset to what's happening now. Unix solved the problem of making computing composable and human-readable through a shell, modular tools, and a file system. Agents solve the same problem for language models: "An agent is the following: it's a language model, and then above that, it's a bash shell. And then the agent has access to the shell and a file system." The state is just files. The implications are profound — you can swap out the underlying model and retain all state, migrate the agent to a different runtime, and instruct the agent to extend itself with new capabilities by writing its own code. "There has never been a widely deployed software system where the thing you're using actually has full introspective knowledge of how it itself works and is able to modify itself like that."

On the death of the browser: Andreessen sees agents gradually displacing the browser as the primary human-computer interface. Once an agent has access to a shell and a browser, "you've got the full capabilities of the browser." The original browser design choice he's most proud of — using text protocols and human-readable HTML — turns out to be prophetic for the AI era. Language models thrive on human-readable text. "The key breakthrough in the browser was the view source option. You could see how it worked, teach yourself how to build new web pages. There's incredible latent power in giving everybody the option to drop down and understand how it's working. I think it's working really well for AI."

On what to build: Andreessen parts ways with AI "purists" who think capabilities alone will determine what gets built. The outside world of 8 billion people making collective decisions is "really complicated and messy" — and the most powerful ways to use agents are more technical than prior software eras. That's not a slowdown, it's a hiring signal: engineers aren't going away, they're becoming the essential operators of the automated enterprise.

**Latent Space：Marc Andreessen 谈浏览器的死亡、Pi + OpenClaw，以及"这次真的不一样"**

**核心洞见：** AI agent 不只是一种新软件——它们是语言模型时代的"Unix shell 时刻"。Marc Andreessen 认为，这套架构现在回头看已经显而易见：LLM + shell + 文件系统 + markdown + cron。

Marc Andreessen 亲历了多次 AI 寒冬——他 1989 年就在用 Lisp 写代码。他认为这一轮 AI 浪潮与以往根本不同的原因是结构性的：我们现在已经确定，神经网络就是正确的架构。这个问题争议了近 70 年，直到最近才尘埃落定。今天构建的一切，都源于 1943 年一篇论文中的原始想法。"这是一场长达八十年的一夜成名，"他说，"ChatGPT 横空出世，o1 出世，OpenClaw 出世——这些都是一夜之间颠覆性的成功，但它们汲取的是八十年来的思想积累。"

在谈到 Pi 和 OpenClaw 的架构突破时，Andreessen 在 1970 年代的 Unix 思维与当下之间画出了一条直线。Unix 通过 shell、模块化工具和文件系统，解决了让计算可组合、可阅读的问题。Agent 为语言模型解决了同样的问题："一个 agent 是这样的：一个语言模型，其上是一个 bash shell，agent 可以访问这个 shell 和文件系统。"状态就是文件。这带来的含义极为深远——你可以替换底层模型而保留所有状态，可以把 agent 迁移到不同的运行时，甚至可以让 agent 通过编写自己的代码来扩展自身能力。"历史上从来没有一个被广泛部署的软件系统，其运行实体本身对自己的运作原理拥有完整的自省能力，并且能够以这种方式修改自己。"

关于浏览器之死：Andreessen 认为，agent 将逐渐取代浏览器，成为人机交互的主界面。一旦 agent 能够访问 shell 和浏览器，"你就拥有了浏览器的全部能力。"他最引以为豪的浏览器原始设计决策——使用文本协议和人类可读的 HTML——在 AI 时代被证明是有先见之明的。语言模型正是在人类可读的文本中如鱼得水。"浏览器最关键的突破是'查看源代码'功能。你能看到它是怎么运作的，从而学会如何创建新网页。赋予每个人随时可以看穿系统运作方式的能力，其中蕴藏着惊人的潜力。我认为这对 AI 同样适用。"

关于建什么：Andreessen 与 AI"纯粹主义者"的观点有所分歧——后者认为能力本身就能决定未来。现实世界有 80 亿人在做集体决策，"真的很复杂、很混乱"——而利用 agent 最有力的方式，比以往的软件时代更需要技术能力。这不是减速信号，而是招聘信号：工程师不会消失，他们将成为自动化企业不可或缺的运营者。
