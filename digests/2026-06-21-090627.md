Going forward: broader eval suites for every system prompt change, soak periods for intelligence-affecting changes, and a new @ClaudeDevs X account for transparent product communication.

**Scaling Managed Agents: Decoupling the brain from the hands**

Anthropic Engineering explains how Managed Agents was architected around a core insight borrowed from OS design: virtualize the components so the interfaces outlast any particular implementation.

The key move was decoupling the "brain" (Claude + harness) from the "hands" (sandboxes and tools) and the "session" (the durable event log). In the original coupled design, everything shared a container — a "pet" that had to be nursed back to health when it failed, and that couldn't be debugged without opening a shell next to user data. Post-decoupling, the container became "cattle": if it fails, the harness catches it as a tool-call error, Claude retries, and a new container is initialized from a standard recipe. The session lives outside both, so a new harness can resume from any point with `wake(sessionId)`.

The security benefit is significant: by ensuring credentials never live in the sandbox where Claude's generated code runs, a prompt injection can't escalate to credential theft. Performance also jumped dramatically: p50 time-to-first-token dropped roughly 60% and p95 dropped over 90%, because containers are now provisioned only if actually needed.
https://www.anthropic.com/engineering/managed-agents

Anthropic Engineering 详解了 Managed Agents 的架构设计，其核心思路借鉴自操作系统设计：将组件虚拟化，使接口的生命周期超越任何具体实现。

关键举措是将"大脑"（Claude + harness）、"双手"（沙盒和工具）以及"会话"（持久化事件日志）三者彻底解耦。在早期的耦合设计中，所有组件共享一个容器——这个容器是个需要精心呵护的"宠物"，一旦故障就要费力恢复，且调试时必须在含有用户数据的环境中开 shell。解耦之后，容器变成了"牲口"：出现故障，harness 将其捕获为工具调用错误，Claude 发起重试，新容器从标准配方初始化即可。会话独立于两者之外，新的 harness 可以随时通过 `wake(sessionId)` 从任意断点恢复。

安全收益同样显著：确保凭证从不出现在 Claude 生成代码运行的沙盒中，提示词注入就无法升级为凭证窃取。性能也大幅提升：p50 首 token 延迟下降约 60%，p95 下降超过 90%——因为容器只在真正需要时才会被创建。
https://www.anthropic.com/engineering/managed-agents

**New in Claude Managed Agents: self-hosted sandboxes and MCP tunnels**

Claude Managed Agents now supports two enterprise-critical features in beta. Self-hosted sandboxes (public beta) let you run agent tool execution inside your own infrastructure or with managed providers — Cloudflare, Daytona, Modal, or Vercel — while Claude's orchestration layer stays on Anthropic's infrastructure. Files and repos never leave your perimeter, and you control compute sizing and the runtime image.

MCP tunnels (research preview, request required) let agents reach private MCP servers — internal databases, APIs, knowledge bases — without exposing them to the public internet. A lightweight gateway you deploy makes a single outbound connection; no inbound firewall rules, no public endpoints, traffic encrypted end-to-end.

Real deployments already running on this: Amplitude's Design Agent on Cloudflare, Clay's GTM engineering agent Sculptor on Daytona, and Rogo's institutional finance analyst agent on Vercel Sandbox.

The three cover the biggest shifts since their last conversation: the coding agent threshold crossing into genuine usefulness, the fracturing of the open-weight model ecosystem, the lab wars between Anthropic and OpenAI, and some genuinely spicy predictions.

The most consequential shift Ari identifies is structural: engineers at AI-native companies are no longer individual contributors — they're now managers of multiple agents running in parallel. "We're really starting to now see the shift of engineers at least kind of almost all moving from ICs to managers of agents." The bottlenecks haven't disappeared; they've moved to code review, understanding, and avoiding what he calls "swap" in the codebase.

On open-weight models, both guests have changed their minds. Ari is now "a lot more bearish on how many open models there will be going forward" — once you've built credibility with an open release, the financial incentive is to close up and sell hosted inference. Rob observes that Meta appears to be pulling back from its open-source strategy, and Chinese labs are keeping their best models proprietary. "I don't think there's a business model, honestly," Rob says of open-source AI.

The spiciest near-term prediction comes from Ari: there's a "very reasonable chance" that Anthropic (or OpenAI) suspends or heavily restricts API access within the next year, purely due to compute constraints — not as a business decision, but because their own first-party products will consume all available inference capacity. Rob seconds the compute theme, arguing that AI's current energy inefficiency is laughable compared to the human brain's 20-watt existence proof, and predicts massive hardware-level breakthroughs in energy efficiency ahead.

On RSI (recursive self-improvement), Ari has shifted: he's now more bullish than six months ago, citing promising internal experiments at Datalogy where agents are doing their own data curation. But he rejects the "runaway" narrative — compute constraints mean RSI will be far slower than the doomsayers predict, and at least ten well-funded labs can work on it simultaneously.

Rob's year-end prediction to watch: Anthropic will become obviously serious about life sciences — setting up wet lab facilities, hiring biologists, and potentially developing their own therapeutic assets. The coding beachhead was the first chapter; biology is the next.
