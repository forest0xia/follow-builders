### Anthropic Engineering

**An update on recent Claude Code quality reports**

Anthropic traced a wave of user-reported quality degradations to three separate, overlapping changes — none of them intentional model regressions. On March 4, Claude Code's default reasoning effort was quietly dropped from high to medium to reduce UI-freezing latency; user feedback pushed them to revert on April 7. On March 26, a bug in a session-clearing optimization caused Claude to reset its memory every single turn instead of just once on resume — making it appear "forgetful and repetitive" — fixed April 10. On April 16, a system prompt change to reduce verbosity backfired on coding quality and was reverted April 20. All three issues are resolved as of v2.1.116. Anthropic says the overlapping changes on different traffic slices is what made the aggregate effect look like broad, inconsistent degradation. Going forward they commit to tighter safeguards before shipping prompt-level changes.
https://www.anthropic.com/engineering/april-23-postmortem

Anthropic 将一系列用户反映的质量下滑追溯到三个独立但相互叠加的变更——没有一个是有意为之的模型退步。3 月 4 日，Claude Code 的默认推理强度从 high 悄悄调低至 medium，以减少界面卡顿；用户反馈后于 4 月 7 日回滚。3 月 26 日，一个 session 清理优化的 bug 导致 Claude 每一轮对话都重置记忆，而不是仅在恢复时重置一次，让 Claude 显得"健忘且重复"；4 月 10 日修复。4 月 16 日，一条旨在减少冗余输出的 system prompt 变更影响了代码质量，4 月 20 日回滚。所有问题已在 v2.1.116 中解决。Anthropic 表示，三个变更作用于不同流量切片且时间错开，导致整体效果看起来像是大范围、不稳定的质量下滑。他们承诺未来在上线 prompt 层面的变更前会设置更严格的防护措施。

**Scaling Managed Agents: Decoupling the brain from the hands**

Anthropic Engineering explains how Claude Managed Agents solves a classic systems design problem: building infrastructure flexible enough for "programs as yet unthought of." The key insight borrows from operating system design — just as OS abstractions like `process` and `file` outlasted the hardware they originally ran on, Managed Agents virtualizes three components of an agent: the **session** (the append-only event log), the **harness** (the loop that calls Claude and routes tool calls), and the **sandbox** (the execution environment). Because these interfaces are stable while implementations underneath can be freely swapped, a harness built for Claude Sonnet 4.5 that added "context resets" to fight context anxiety worked fine — until the same fix became unnecessary dead weight on Claude Opus 4.5. Managed Agents lets these assumptions be updated without rebuilding everything. The result is a hosted service on the Claude Platform that runs long-horizon agents through stable abstractions designed to outlast any specific model generation.
https://www.anthropic.com/engineering/managed-agents

**Anthropic Engineering 解析 Managed Agents 架构：将"大脑"与"双手"解耦**

Anthropic Engineering 分享了 Claude Managed Agents 如何解决一个经典的系统设计难题：如何构建对"尚未存在的程序"也足够灵活的基础设施。核心思路借鉴自操作系统设计——就像 `process`、`file` 这样的 OS 抽象层比其底层硬件活得更久，Managed Agents 将 agent 的三个核心组件虚拟化：**session**（只追加的事件日志）、**harness**（调用 Claude 并路由工具调用的循环）和 **sandbox**（执行环境）。由于这些接口保持稳定而底层实现可以自由替换，一个为 Claude Sonnet 4.5 的"context 焦虑"添加 context reset 的 harness，到了 Claude Opus 4.5 上就变成了多余的死重。Managed Agents 让这些假设可以随时更新，而无需重建整个系统。最终产品是 Claude Platform 上一项托管服务，通过设计寿命长于任何具体模型世代的稳定抽象来运行长时程 agent 任务。
https://www.anthropic.com/engineering/managed-agents

**New in Claude Managed Agents: self-hosted sandboxes and MCP tunnels**

Claude Managed Agents now supports two major enterprise features. The most provocative thread concerns the future of the public API.
