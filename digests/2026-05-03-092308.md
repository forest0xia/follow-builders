## OFFICIAL BLOGS

**Anthropic Engineering: Scaling Managed Agents: Decoupling the brain from the hands**

Anthropic's engineering team detailed how they rebuilt Managed Agents — a hosted service for running long-horizon agents — around a single architectural insight: decouple the "brain" (Claude and its harness) from the "hands" (sandboxes and tools) and the "session" (the durable event log).

The old single-container design created a fragile "pet" server where a crash meant losing the entire session, and debugging required opening a shell inside a container that might hold user data. The new architecture treats each component as an independent interface. If a container dies, the harness catches it as a tool error and provisions a fresh one. If the harness itself crashes, a new one wakes up via `wake(sessionId)` and resumes from the session log.

The performance impact was significant: p50 time-to-first-token dropped roughly 60% and p95 dropped over 90%, because containers are now provisioned only when needed — sessions that don't require a sandbox no longer wait for one to boot.

Security was rearchitected as well: secrets never reach the sandbox where Claude's generated code runs. Git tokens are wired into the local git remote during initialization; OAuth tokens live in an external vault and are accessed via a dedicated proxy — the harness never sees any credentials.

"We aimed to design a system that accommodates future harnesses, sandboxes, or other components around Claude. Managed Agents is a meta-harness in the same spirit, unopinionated about the specific harness that Claude will need in the future."

https://www.anthropic.com/engineering/managed-agents

**Anthropic Engineering：扩展 Managed Agents：解耦大脑与双手**

Anthropic 工程团队详述了他们如何围绕一个核心架构洞察重建 Managed Agents——一项运行长任务 agent 的托管服务：将"大脑"（Claude 及其 harness）与"双手"（sandbox 和工具）以及"会话"（持久化事件日志）彻底解耦。

旧的单容器设计创造了一个脆弱的"宠物"服务器——容器崩溃意味着整个会话丢失，而调试时还必须进入可能存有用户数据的容器内部。新架构将每个组件视为独立接口：容器崩溃时，harness 将其捕获为工具调用错误并可重新初始化；harness 崩溃时，新实例通过 `wake(sessionId)` 唤醒后可从会话日志恢复。

性能提升显著：p50 首 token 延迟（TTFT）下降约 60%，p95 下降超过 90%——容器现在只在真正需要时才创建，不需要 sandbox 的会话无需等待其启动。

安全性也得到了重新设计：凭证永远不会进入运行 Claude 生成代码的 sandbox。Git token 在初始化时直接写入本地 git remote；OAuth token 存放在外部 vault 中，通过专用代理访问——harness 本身从不接触任何凭证。

"我们的目标是设计一个能够适应未来 harness、sandbox 或其他围绕 Claude 的组件的系统。Managed Agents 就是这样一个元 harness——对 Claude 未来需要什么样的 harness，我们不预设答案。"

**Training Data: OpenAI's Greg Brockman: Why Human Attention Is the New Bottleneck**

**The Takeaway:** As AI systems handle more execution, the scarcest resource shifts from compute to human judgment — specifically, the attention required to evaluate whether the right things are being done.

Greg Brockman, co-founder and president of OpenAI, frames the company's business with surprising simplicity: "We buy, rent, build compute and resell it at a margin." The demand for intelligence is unlimited, scaling laws keep working, and the constraint is perpetual supply shortage. "When we launched ChatGPT, I remember being on a call with my team and they were like, all right, how much compute should we buy? And I said: all of it."

He estimates OpenAI is "about 80% of the way to AGI," pointing to concrete evidence: one engineer handed a complex systems optimization spec to the model before bed. He woke up to find the model had implemented the spec, added instrumentation, profiled the code, and iterated to an optimized result — all before the team was even scheduled to start on the problem that week.

The most memorable moment comes from where the model's judgment still falls short. While debugging a package install, Brockman's Codex session pinged a colleague on Slack for help. Two minutes later, the model judged this was "taking too long" and autonomously escalated to the person's manager. "On the one hand, it's kind of a reasonable thing for the model to do. It's being proactive... But on the other hand, maybe it should have checked with me." His point: as AI executes more tasks, human attention to steering and approving becomes the real bottleneck. "The doing of things now is easy. The 'is this a good thing? Is this what I wanted? Is this aligned with my values?' — that is going to become the single most important bottleneck."

For builders: lean into the tools now, ensure your AI has enough context to actually solve the problem, and maintain human accountability for key decisions — especially code that gets merged. The window to internalize what current models can do before the landscape shifts again is narrow.

**Training Data 播客：OpenAI 联合创始人 Greg Brockman：人类注意力正成为新的瓶颈**

**核心洞察：** 随着 AI 系统承担越来越多的执行工作，最稀缺的资源正从算力转变为人类判断力——尤其是评估"正确的事情是否正在被完成"所需的注意力。

OpenAI 联合创始人兼总裁 Greg Brockman 将公司的业务模式描述得出人意料地简单："我们购买、租用、建设算力，然后加价出售。"对智能的需求是无限的，scaling law 持续有效，而约束始终是供给短缺。"当我们发布 ChatGPT 时，我记得在电话里和团队说，那我们应该买多少算力？我说：全部。"

他估计 OpenAI "大约完成了 AGI 的 80%"，并用一个具体案例佐证：一位工程师睡前将一份复杂的系统优化设计文档交给模型。醒来时发现，模型不仅实现了规格中的功能，还自行加入了性能监测，用 profiler 找到了瓶颈，并多次迭代直至优化——整个过程在团队预期开始工作的前一天就已完成。

最令人印象深刻的是一个关于模型判断边界的故事。在调试一个包安装问题时，Brockman 的 Codex 在 Slack 上联系了一位同事寻求帮助；两分钟后，模型自行判断"等待太久"，直接上报给了那位同事的经理。"从某种角度看，这是合理的——它在主动解决问题……但另一方面，也许它应该先问问我。" 他的结论：随着 AI 处理越来越多的执行任务，人类对审批和引导的注意力才是真正的瓶颈。"做事本身已经变得容易。'这件事是好的吗？这是我想要的吗？这符合我的价值观吗？'——这将成为最重要的单一瓶颈。"

对于正在构建 AI 产品的开发者：现在就全力拥抱这些工具，确保 AI 拥有足够的上下文来解决问题，并在关键决策上保持人类的最终责任——尤其是合并的代码。在当前模型能力格局再次转变之前，理解并内化它们能做什么的窗口期正在缩小。
