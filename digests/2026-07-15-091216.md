**Training Data: Anthropic's Katelyn Lesse & Angela Jiang — Building an Ecosystem, not a Walled Garden**
https://www.youtube.com/watch?v=vPnVTHYplrQ

**The Takeaway:** Anthropic's platform team thinks in three layers — knowledge, execution, coordination — and they're now building the coordination layer: composable "strategies" that give tokens a distinct *job* to do, not just a direction to point.

Katelyn Lesse and Angela Jiang lead Anthropic's externally-facing Claude APIs and the internal platform powering Anthropic's own products. Their framing is refreshingly architectural: most harness engineering debates are focused on the wrong layer. Prompt caching, context window hygiene, tool call routing — those are table stakes. And increasingly, the models are so steerable that the scaffolding once needed to guide them from A to B can simply be deleted. The real frontier is a layer higher: what job do you give each token?

Their most distinctive concept is "strategies" — meta-harnesses sitting above execution that assign tokens distinct roles: adviser, executor, grader, dreamer. "If tokens aren't really fungible and you need to give them different jobs," Jiang explains, "you wanna start composing these kind of orchestrated strategies." They see this as the natural next form factor — just as agents replaced pure chat, something will displace generic single-agent loops.

On open ecosystems vs. walled gardens, their position is clear: Anthropic doesn't care which infrastructure you use (they launched self-hosted sandboxes with Modal, Vercel, Cloudflare, and Amazon micro-VMs), as long as you build to their architectural opinions. Claude Tag — which many dismissed as just a Slack bot — is their proof-of-concept for the iceberg principle: the interface is the visible tip; the context engineering and proactivity architecture underneath is the real product.

What excites them most from the customer base right now: companies wrapping agents as MCP servers so different model families can call each other, and healthcare companies using computer use to connect to legacy systems that don't have APIs.

---

**The Takeaway（要点）：** Anthropic 的平台团队以三层架构来思考问题——知识层、执行层、协调层——而他们现在正在构建的正是协调层：可组合的"strategies"（策略），为每个 token 分配明确的**工作角色**，而不仅仅是指明方向。

Katelyn Lesse 和 Angela Jiang 负责 Anthropic 对外的 Claude API 以及支撑 Anthropic 内部产品的平台层。她们的框架令人耳目一新：大多数关于 harness 工程的争论都聚焦在错误的层次。Prompt caching、context window 管理、tool call 路由——这些都是基础配置。而且随着模型的可控性越来越强，过去用来引导模型从 A 到 B 的脚手架其实可以直接删掉。真正的前沿在更高一层：你给每个 token 分配什么**工作**？

她们提出的最核心概念是"strategies"——坐落于执行层之上的元 harness，为 token 分配不同角色：顾问、执行者、评分者、创想者。"如果 token 并非完全可替代，你需要给它们不同的工作，" Jiang 解释道，"你会想要开始组合这种精心编排的策略。"她们认为这是 AI 应用形态演进的自然下一步——正如 agent 取代了纯对话，某种新形态将取代通用单 agent 循环。

在开放生态与封闭花园的问题上，她们的立场清晰：Anthropic 不在乎你用哪家的基础设施（她们已与 Modal、Vercel、Cloudflare 和亚马逊 micro-VM 合作推出自托管沙盒），只要你遵循她们的架构理念构建即可。Claude Tag 曾被许多人嘲为"不过是个 Slack 机器人"——对她们来说，这恰恰是冰山原则的最佳佐证：界面是水面之上的可见部分；其下的 context 工程和主动性架构才是真正的产品。

目前让她们最兴奋的客户案例：有公司将 agent 包装为 MCP server，让不同模型家族的 agent 可以互相调用；还有医疗公司借助 computer use 连接那些没有 API 的遗留系统。
